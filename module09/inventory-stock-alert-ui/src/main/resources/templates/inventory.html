UI-10: Inventory Stock Alert using Kafka

Kafka Flow

Inventory updates are published to inventory-events-topic. The consumer checks the stock quantity. If stock is low, a new alert event is published to stock-alert-topic. 

Inventory UI
    ↓
inventory-events-topic
    ↓
InventoryEventConsumer
    ↓
if currentStock <= reorderThreshold
    ↓
stock-alert-topic
    ↓
StockAlertConsumer
    ↓
Dashboard

Important Clarification

Kafka is responsible for storing and delivering events through topics.

Kafka is not deciding whether stock is low. The low-stock check is written in Java application code.

The dashboard tables are also application-level views. They are stored in memory using Java collections. Kafka's actual event log is inside topic partitions.
