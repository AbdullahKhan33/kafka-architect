Kafka Transaction Concept

Topic 1: ecommerce-orders-topic

Topic 2: ecommerce-inventory-topic

One button publishes to both topics inside one Kafka transaction.

Success Case

BEGIN TRANSACTION

orders-topic
  ORDER_PLACED

inventory-topic
  INVENTORY_RESERVED

COMMIT

Failure Case

BEGIN TRANSACTION

orders-topic
  ORDER_PLACED

Simulated Failure

ABORT

No committed event should be visible

Key Learning

  • Kafka can publish atomically to multiple topics.
  • Either all events are committed or none are visible.
  • Consumers with read_committed only read committed records.
  • This prevents partial event publishing.
